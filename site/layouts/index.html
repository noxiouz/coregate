Rust coredump collection for Linux

Collect coredumps. Keep the hot path short.

Coregate captures Linux coredumps through `core_pattern`, legacy socket delivery, and protocol socket delivery. It extracts metadata worth querying, stores dumps with explicit policy, and validates the real kernel path inside QEMU.

Why Coregate

Focused on the real coredump path.

The tool is opinionated where it should be: fast intake first, explicit kernel mode support, queryable metadata, and end-to-end validation against real guests instead of mocked inputs.

Three ingress modes

`handle` for classic `core_pattern`, `serve-legacy` for `@/path.sock`, and `serve` for `@@/path.sock` protocol mode.

Fast-path aware

Cheap metadata first, core drain before expensive enrichment, and explicit handling of dumpability, storage reserve, and rate limiting.

Useful metadata

PID and namespace identity, executable and build information, runtime classification, cgroup context, and dump outcome.

Storage with policy

Sparse or compressed core files, free-space protection, local JSONL records, and optional SQLite indexing.

Kernel-version aware

`coregate setup` refuses unsupported socket modes based on the running kernel version instead of printing invalid setup commands.

VM-backed validation

QEMU scenarios exercise real crashes in Debian guests with explicit kernel selection for `@` and `@@` protocol coverage.

How It Works

Three steps, one binary, real kernel integration.

1

Choose the kernel mode

Use `handle` for wide compatibility, `serve-legacy` for Linux 6.16+, or `serve` for the 6.19+ protocol path.

coregate setup handle
coregate setup server-legacy
coregate setup server
2

Store the right artifacts

Configure compression, sparse writes, SQLite indexing, and a minimum free-space reserve to prevent dump storms from filling disks.

{
  "core": { "compression": "zstd" },
  "metadata_sqlite": "/var/lib/coregate/records.sqlite"
}
3

Validate it in a VM

Run scenario-based guest tests that trigger real crashes and assert on the produced metadata and stored core files.

cargo run -p xtask -- vmtest run --scenario all
cargo run -p xtask -- vmtest run --scenario server-segv

Commands

Use the binary directly.

Handle mode

sudo coregate setup handle --apply

Classic `core_pattern` pipe-helper integration.

Legacy socket mode

sudo coregate setup server-legacy --apply

Enables `@/path.sock` on Linux 6.16 and newer.

Protocol socket mode

sudo coregate setup server

Enables `@@/path.sock` on Linux 6.19 and newer.

VM scenarios

cargo run -p xtask -- vmtest run --scenario thread-crash

Exercises the guest path end to end.

Install

Build locally and start with the docs.

Coregate is currently built from source. The docs cover setup, configuration, and VM-backed testing in one place.

cargo build -p coregate
cargo test -p coregate
