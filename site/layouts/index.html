Rust coredump collection for Linux

Collect coredumps. Keep the hot path short.

Coregate captures Linux coredumps through core_pattern, legacy socket delivery, and protocol socket delivery. It extracts metadata worth querying, stores dumps with explicit policy, and can enrich records with BPF-captured user stacks.

Read the Quickstart Documentation

$ sudo coregate-bpf install --force
$ sudo coregate serve --apply-sysctl

Why Coregate

Focused on the real coredump path.

The tool is opinionated where it should be: fast intake first, explicit kernel mode support, queryable metadata, optional BPF stack enrichment, and storage policy that does not leave disk pressure to chance.

Three ingress modes

handle for classic core_pattern, serve-legacy for @/path.sock, and serve for @@/path.sock protocol mode.

Fast-path aware

Cheap metadata first, core drain before expensive enrichment, and explicit handling of dumpability, storage reserve, and rate limiting.

Useful metadata

PID and namespace identity, executable and build information, runtime classification, cgroup context, and dump outcome.

Storage with policy

Sparse or compressed core files, free-space protection, local JSONL records, and optional SQLite indexing.

BPF stack capture

Optional do_coredump tracing stores raw user frames, adds best-effort live blazesym symbols, and records normalized offsets for later symbolization.

Kernel-version aware

coregate setup refuses unsupported socket modes based on the running kernel version instead of printing invalid setup commands.

How It Works

Three steps, one binary, real kernel integration.

1

Choose the kernel mode

Use handle for wide compatibility, serve-legacy for Linux 6.16+, or serve for the 6.19+ protocol path.

coregate setup handle
coregate setup server-legacy
coregate setup server
2

Set storage and policy

Configure compression, sparse writes, SQLite indexing, ordered overrides, and a minimum free-space reserve to prevent dump storms from filling disks.

{
  "default": {
    "core": { "compression": "zstd" },
    "metadata_sqlite": "/var/lib/coregate/records.sqlite"
  }
}
3

Optionally enable BPF stacks

Install the pinned tracer and use a coregate build with the bpf feature if you want crash records to include captured user frames and symbolization metadata.

sudo coregate-bpf install --force
sudo coregate debug-bpf-stats --json  # bpf-enabled build

Commands

Use the binary directly.

Handle mode

sudo coregate setup handle --apply

Classic core_pattern pipe-helper integration.

Legacy socket mode

sudo coregate setup server-legacy --apply

Enables @/path.sock on Linux 6.16 and newer.

Protocol socket mode

sudo coregate serve --apply-sysctl

Enables @@/path.sock on Linux 6.19 and newer.

BPF tracer

sudo coregate-bpf install --force

Loads and pins the do_coredump stack tracer.

Install

Build locally and start with the docs.

Coregate is currently built from source. The docs cover installation, kernel setup, optional BPF stack capture, and configuration in one place.

# Cargo
cargo build -p coregate
cargo build -p coregate-bpf

# Or Bazel
bazel build //crates/coregate:coregate
