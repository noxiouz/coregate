Rust coredump collector for Linux

Fast crash intake, real kernel modes, and VM-backed validation.

Coregate captures coredumps through `core_pattern` or socket-based kernel delivery, extracts metadata worth querying, stores dumps with sane policies, and validates behavior inside QEMU instead of pretending unit tests are enough.

Read the Quickstart View on GitHub 
sudo coregate setup handle --apply
sudo coregate setup server-legacy --apply
sudo coregate setup server --apply

cargo run -p xtask -- vmtest run --scenario all

Kernel-aware ingress

`handle` for classic `core_pattern`, `serve-legacy` for `@/path.sock`, and `serve` for `@@/path.sock` protocol mode.

Useful metadata

Process identity, executable details, build IDs, cgroup context, dumpability, runtime classification, and storage outcome.

Practical storage

Sparse or compressed core storage, free-space protection, JSONL output, and optional SQLite indexing.

Kernel modes

Understand `handle`, `serve-legacy`, and `serve`, and how `@` and `@@` map to different socket protocols.

Configuration

JSON parsed into protobuf types, storage policy, metadata sinks, and rate-limit configuration.

Testing

QEMU-backed crash scenarios with Debian guests, explicit kernel selection, and integration tests that hit real coredump paths.
